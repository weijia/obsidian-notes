# Obsidian Notes 需求文档

## 存储后端
- 支持 WebDAV 和 Gitee 两种存储后端
- 配置信息持久化到 localStorage，刷新后自动重连

## 文件列表
- 按最后修改时间降序排列（默认），目录优先于文件
- 默认显示详细信息（文件大小、修改时间）
- 隐藏以 `.` 开头的文件和目录（如 `.git`）
- 排序方式（名称/时间/类型/大小）可切换，设置保存到 localStorage
- 支持简洁/详细两种显示模式切换
- 自动连接期间显示 Loading，不显示连接错误
- 记住最后打开的目录路径，刷新后恢复

## 编辑器
- TipTap 富文本编辑器，支持 Markdown
- 工具栏：标题(H1-H3)、粗体、斜体、删除线、行内代码、列表、引用、代码块、链接、水平线、表格
- 表格操作：插入自定义行列数、增删行列、删除表格
- **未修改时保存按钮禁用（灰色、不可点击）**
  - 加载文件时用 turndown 回转值作为内容基准，避免 markdown↔HTML 转换差异导致误判
  - 保存成功后更新基准，按钮重新变灰

## 布局
- 桌面端（>=768px）：左右分栏，左侧文件列表 + 右侧编辑器同时显示，无底部导航
- 移动端（<768px）：单面板 + 底部导航栏切换文件列表/编辑器
- 窗口 resize 时自动切换布局

## 导航
- 点击文件进入编辑时，记住当前目录路径
- 移动端点击文件进入编辑时 pushState，浏览器返回按钮回到文件列表
- 点击底部"文件"按钮恢复到之前的目录

## 版本与更新
- 顶部"笔记"旁显示版本号和东八区构建时间
- PWA 自动更新检测（vite-plugin-pwa + 自定义 Service Worker）
  - `registerType: 'prompt'` 不自动激活，等待用户操作
  - `injectManifest` 策略 + 自定义 `sw.js`：预缓存 + 清理旧缓存 + 监听 `SKIP_WAITING`
  - 页面加载后 3 秒首次检测，每 5 分钟轮询 `reg.update()`
  - SW `updatefound` → `statechange` → `installed` 时显示"新版本可用"提示
  - 点击刷新发送 `SKIP_WAITING` 消息，新 SW 激活后自动 `location.reload()`
  - `updateAvailable` 标志去重，避免重复弹出
